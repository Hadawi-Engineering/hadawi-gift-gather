Main Components
===============

1. Collection Page System
   • /collect/[id].html
   • Fetches occasion data from Firestore
   • Displays gift details
   • Handles contribution form
   • Integrates with ClickPay

2. Firebase Cloud Functions
   • initiateClickPayPayment - Creates payment session
   • clickPayWebhook - Processes payment callbacks
   • verifyPaymentStatus - Verifies payment completion

3. Firestore Database
   Collections:
   • occasions/ - Gift occasions
   • payments/ - Payment transactions

4. Security & Rules
   • firestore.rules - Database access control
   • Server-side payment processing
   • Environment variable protection


Quick Start Commands
====================

Setup:
$ firebase login
$ firebase use --add
$ firebase deploy

Development:
$ firebase emulators:start

Deployment:
$ firebase deploy                    # Deploy everything
$ firebase deploy --only hosting     # Deploy website only
$ firebase deploy --only functions   # Deploy functions only
$ firebase deploy --only firestore   # Deploy rules & indexes

Monitoring:
$ firebase functions:log             # View function logs

Environment Variables Required
==============================

Client-side (Firebase - Safe to expose):
✓ VITE_FIREBASE_API_KEY
✓ VITE_FIREBASE_AUTH_DOMAIN
✓ VITE_FIREBASE_PROJECT_ID
✓ VITE_FIREBASE_STORAGE_BUCKET
✓ VITE_FIREBASE_MESSAGING_SENDER_ID
✓ VITE_FIREBASE_APP_ID
✓ VITE_FIREBASE_MEASUREMENT_ID

Server-side (ClickPay - Must be secret):
⚠️  CLICKPAY_SERVER_KEY (via Firebase Functions config)
⚠️  CLICKPAY_PROFILE_ID (via Firebase Functions config)
⚠️  CLICKPAY_API_URL (via Firebase Functions config)
